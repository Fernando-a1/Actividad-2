Tema 1: Completa la siguiente tabla

Concepto Descripción Concepto Descripción
Tabla en HTML Una tabla en HTML se utiliza para organizar y mostrar datos en filas y columnas. Se construye utilizando etiquetas como <table>, <tr>, <td>, y <th>. Formulario en HTML Un formulario en HTML permite a los usuarios ingresar y enviar datos al servidor. Utiliza etiquetas como <form>, <input>, <textarea>, <select>, entre otras.
Etiqueta <table> Define una tabla en HTML. Es el contenedor principal que agrupa todas las filas (<tr>) y celdas (<td> y <th>). Etiqueta <form> Define un formulario en HTML. Agrupa los diferentes elementos de entrada como campos de texto, botones, casillas de verificación, etc., que el usuario puede interactuar.
Atributos de <table> Incluyen border, cellspacing, cellpadding, width, height, align, entre otros, que controlan la apariencia y el comportamiento de la tabla. Etiqueta <input> Crea un campo de entrada donde el usuario puede ingresar datos. Puede ser de varios tipos como texto, contraseña, radio, checkbox, botón, etc., especificados con el atributo type.
Etiqueta <tr> Define una fila en una tabla. Cada <tr> puede contener múltiples celdas (<td> o <th>). Etiqueta <label> Asocia una etiqueta de texto con un elemento de formulario, mejorando la accesibilidad. Se utiliza la etiqueta <label> con el atributo for que corresponde al id del elemento de entrada.
Atributo tr No existe un atributo específico llamado tr. Probablemente se refiere a los atributos que pueden usarse en la etiqueta <tr>, como align, valign, bgcolor, etc. Etiqueta <select> Crea un menú desplegable en un formulario, permitiendo al usuario seleccionar una opción de una lista predefinida. Utiliza la etiqueta <option> para cada opción disponible.
Etiqueta <td> y <th> <td> define una celda de datos en una tabla, mientras que <th> define una celda de encabezado, que generalmente se muestra en negrita y centrada. Etiqueta <textarea> Crea un área de texto multilínea para que el usuario ingrese información extensa. Se utiliza principalmente para comentarios, descripciones, etc.
Atributos de <td> y <th> Incluyen colspan, rowspan, headers, scope, align, valign, entre otros, que permiten fusionar celdas y definir su alineación y alcance dentro de la tabla. Etiqueta <fieldset> y <legend> <fieldset> agrupa elementos relacionados en un formulario, y <legend> proporciona un título o descripción para el grupo, mejorando la organización y accesibilidad.
Tablas avanzadas Incluyen características como tablas anidadas, uso de <caption>, combinación de celdas con colspan y rowspan, y estilos avanzados con CSS para mejorar la presentación. Etiqueta <meter> Representa una medida escalar dentro de un rango conocido, como indicadores de progreso, niveles de batería, etc. Es una forma semántica de mostrar datos cuantitativos.
Etiqueta <thead> Define el encabezado de una tabla, agrupando una o más filas de encabezado (<tr>). Mejora la accesibilidad y permite aplicar estilos específicos al encabezado. Etiqueta <progress> Representa el progreso de una tarea en curso. Muestra una barra de progreso que puede ser determinista (con valor) o indeterminada (sin valor específico).
Etiqueta <tfoot> Define el pie de una tabla, agrupando una o más filas que generalmente contienen resúmenes o totales. Ayuda en la organización y en la repetición del pie en tablas largas. Etiqueta <datalist> Proporciona una lista de opciones predefinidas para un campo de entrada (<input>), permitiendo sugerencias automáticas a medida que el usuario escribe.
Etiqueta <tbody> Agrupa el cuerpo de una tabla, separándolo del encabezado (<thead>) y del pie (<tfoot>). Facilita la manipulación y el estilizado del contenido principal de la tabla. Etiqueta <summary> Proporciona un resumen o una descripción para el contenido de un elemento <details>, mejorando la accesibilidad y la experiencia del usuario al interactuar con contenido desplegable.

Tema 2: Define los elementos que componen la estructuración de una página web

Elementos de Estructuración

  • Contenedor (Wrapper): Es un elemento que envuelve todo el contenido de la página, proporcionando una estructura general y facilitando la aplicación de estilos CSS globales. Ayuda a centrar el contenido y a definir límites visuales.
  • Cabecera (Header): Sección ubicada generalmente en la parte superior de la página. Contiene elementos como el logotipo, el título del sitio, y el menú de navegación principal. Proporciona una primera impresión y guía para el usuario.
  • Contenido (Content): Área principal de la página donde se presenta la información relevante, artículos, imágenes, videos y otros tipos de contenido. Es el núcleo de la página y lo que los usuarios buscan.
  • Menú (Menu): Elemento de navegación que permite a los usuarios moverse entre las diferentes secciones o páginas del sitio web. Puede estar ubicado en la cabecera, en una barra lateral o en el pie de página.
  • Pie (Footer): Sección ubicada en la parte inferior de la página. Suele contener información adicional como derechos de autor, enlaces a políticas de privacidad, términos de uso, información de contacto y enlaces secundarios.
  • Lateral (Sidebar): Área ubicada a uno de los lados de la página (izquierda o derecha). Puede contener widgets, enlaces relacionados, anuncios, menús adicionales o cualquier otro contenido complementario.
  • Etiqueta <div>: Elemento genérico de contenedor en HTML que no tiene significado semántico propio. Se utiliza para agrupar otros elementos y aplicar estilos o scripts específicos. Es fundamental para la maquetación y estructuración del contenido.

Tema 3: Identifica y describe los elementos que componen la estructuración semántica de una página web

Elementos de Estructuración Semántica

  • <header>: Define la cabecera de una sección o de toda la página. Generalmente contiene el logotipo, el título del sitio, y el menú de navegación principal. Puede repetirse en diferentes secciones de la página.
  • <nav>: Representa una sección de navegación, como menús principales, barras laterales de navegación o cualquier otro grupo de enlaces que facilitan el movimiento dentro del sitio web.
  • <main>: Contiene el contenido principal único de la página, excluyendo elementos que se repiten en otras páginas como cabeceras, pies de página o menús de navegación. Solo debe haber un <main> por página.
  • <section>: Agrupa contenido temáticamente relacionado dentro de una página. Cada <section> debe estar encabezado por un <h1>-<h6>. Es útil para dividir el contenido en partes lógicas y manejables.
  • <article>: Representa un contenido independiente y autocontenible que podría ser distribuido de manera independiente, como una entrada de blog, una noticia o un comentario de usuario.
  • <aside>: Contiene contenido secundario o complementario al contenido principal, como barras laterales, enlaces relacionados, anuncios o widgets. Se utiliza para información adicional que no es esencial para la comprensión del contenido principal.
  • <footer>: Define el pie de una sección o de toda la página. Suele incluir información de contacto, derechos de autor, enlaces a políticas de privacidad, y otros enlaces secundarios.
  • <figure> y <figcaption>: <figure> se utiliza para agrupar contenido multimedia como imágenes, diagramas, ilustraciones, etc., y <figcaption> proporciona una leyenda o descripción para ese contenido.
  • <details> y <summary>: <details> crea un widget interactivo que el usuario puede abrir y cerrar para ver más información. <summary> define el título visible del widget, que se muestra cuando el contenido está cerrado.
  • <mark>: Resalta un texto para llamar la atención del usuario sobre él. Útil para destacar palabras clave o información importante.
  • <time>: Representa una fecha u hora específica. Puede ser utilizada para indicar fechas de eventos, publicaciones, modificaciones de contenido, etc.
  • <address>: Proporciona información de contacto sobre el autor o el propietario del documento o artículo. Suele incluir direcciones, correos electrónicos, números de teléfono, etc.
  • <dialog>: Define un cuadro de diálogo o ventana emergente que puede ser modal o no modal. Utilizado para interacciones como confirmaciones, formularios o mensajes.

Beneficios de la Estructuración Semántica

  • Mejora la accesibilidad: Los lectores de pantalla y otras tecnologías de asistencia pueden interpretar mejor el contenido.
  • Optimización para motores de búsqueda (SEO): Los motores de búsqueda comprenden mejor la estructura y relevancia del contenido, mejorando el posicionamiento.
  • Mantenimiento y legibilidad del código: Facilita la comprensión y el mantenimiento del código por parte de otros desarrolladores.
  • Consistencia y estandarización: Promueve el uso de estándares web, asegurando que el contenido sea consistente a través de diferentes navegadores y dispositivos.
